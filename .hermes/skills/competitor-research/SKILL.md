# Competitor Research

Use this skill to create a competitor and ad-angle analysis.

## Inputs

- `clients/<client>/outputs/icp-brief.md`
- `clients/<client>/inputs/competitor-list.md`
- Foreplay API results when available
- `clients/<client>/inputs/ad-example-library.md` fallback

## Output

Write `clients/<client>/outputs/competitor-analysis.md`.

## Sourcing Priority

1. Foreplay Discovery / Spyder API results
2. Public competitor websites
3. Manual examples in `ad-example-library.md`
4. Clearly labeled assumptions

## Required Sections

- Competitor table
- Foreplay ad research summary
- Positioning patterns
- Offer patterns
- Hook patterns
- Creative patterns
- Landing page patterns
- White space
- What to steal
- What to avoid
- Unknowns and assumptions

## Rules

- Do not claim competitor performance unless the source provides it.
- Use "appears to" or "suggests" for interpretation.
- Save normalized ad examples back into `ad-example-library.md` when possible.
